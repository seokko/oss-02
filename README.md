## 현재 까지 배운내용
### github 저장소 구축 방법
* repositories
* branch
### github 용어
* Commit:프로젝트의 변화를 만드는 것
* Push:온라인 respository에 commit을 발행
* Pull(or update, or fetch):프로젝트의 카피본에 다른사람의 commit을 가져오는 것
* Commit message(or log message):각 commit에 대한 요약 설명
* Repositoriy: 변화를 저장하고 발행된 것에 대한 데이터베이스
* Clone: 프로젝트의 복사본을 생성하여 자신의 개발장소를 획득
* Checkout:clone과 비슷하나, working 카피본만 획득
* Working copy(or working files):개발자의 개인 디렉토리 트리
* Revision, change, changeset, or commit: 특정 시점에서 변화된 파일
* Diff:변화의 텍스트적 표현
* Branch:프로젝트의 카피, 하지만 고립되어 있는 형태(main branch, main line, master 등)
* Merge(or port):변화를 어떤 branch에서 다른branch로 이동시킴
* Conflict:두 사람이 코드 내 하나의 구간을 동시에 수정하는 경우
* Revert or reversion:이미 수행된 commit을 undo
* Lock:특정 파일이나 디렉토리에 대한 변화를 제외하는 방법
